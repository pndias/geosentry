# GeoSentry - Genesis Documentation (v1.0.0)

## 📅 Date: March 09, 2026
## 🏛️ Purpose and Philosophy
GeoSentry is a "Data Shield" designed for the Global South. Its mission is to use Artificial Intelligence to save lives through autonomous geopolitical monitoring, prioritizing popular sovereignty, civilian security, and the working-class perspective in conflict scenarios.

## 🛠️ What was built today:

### 1. Modular Architecture (Microservices & Clean Architecture)
- **`src/core`**: Rigorous data models (Pydantic).
- **`src/api`**: Refactored backend with isolated routes (`routers`) and business logic (`services`).
- **`src/agents`**: AI Agent orchestration (CrewAI) focused on OSINT via Serper.
- **`frontend`**: Modern SPA in React + TypeScript, isolated components and Custom Hooks.

### 2. GeoSentry Dashboard v1.1
- **Identity**: Updated to "Global South Sentinel" (Avatar: GS).
- **Visualization**: Interactive map with dynamic markers by impact (1-5) and color-coded categories.
- **Responsiveness**: Adaptive layout with event sidebar and statistics.

### 3. Data Persistence (SQLite Seed)
- Initial database implementation via **SQLAlchemy** (`sqlite:///./geosentry.db`).
- Seed script (`seed_db.py`) generating **100 realistic and randomized occurrences** focused on the Global South for load testing and map rendering.
- API fully connected and reading data from the real database.

### 4. Governance and Transparency
- **MANIFESTO.md**: Principles of technological sovereignty, life protection, and working-class defense.
- **ROADMAP.md**: 4-phase plan for the system's future, including NLP analysis.

### 5. Stability and Fixes (Frontend)
- **TypeScript and Build**: Compiler configuration fixes (`tsconfig.json`) enabling `esModuleInterop` and `allowSyntheticDefaultImports` to stabilize React imports.
- **Code Formatting**: Replaced Python-style comments (`#`) with JavaScript-style comments (`//`) in `.ts` and `.tsx` files to prevent Babel/Vite parser breaks.
- **Copyright**: Legal protection enabled with MIT License and authorship attribution to Pablo Dias, applied safely (without breaking compilation).
