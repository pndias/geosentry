# GeoSentry - Docker Swarm & Multi-Environment Deploy (v1.2.0)

## 📅 Date: March 18, 2026
## 🚀 What's New:

### 1. Containerized Microservices Architecture
Every service now runs as an independent Docker container, fully isolated:

| Container | Image | Purpose |
| :--- | :--- | :--- |
| `db` | postgis/postgis:15 | PostgreSQL + PostGIS database |
| `api` | geosentry/api | FastAPI backend (2 replicas in Swarm) |
| `agents` | geosentry/agents | CrewAI geopolitical intelligence crew |
| `ingestion` | geosentry/ingestion | Pydantic data extraction processor |
| `frontend` | geosentry/frontend | React SPA via nginx reverse proxy (2 replicas in Swarm) |
| `seed` | geosentry/seed | One-shot database seeder |
| `ollama` | ollama/ollama | Local LLM server (llama3.2:3b) |

### 2. Docker Swarm Orchestration
- `docker-compose-swarm.yml` — Full stack definition with overlay network, replica counts, restart policies, and rolling update config.
- API and Frontend scaled to 2 replicas with Swarm ingress load balancing.
- Seed container runs once (restart policy: `none`).

### 3. Dual Deployment Scripts

#### `run-macos.sh` — macOS Local Development
- Checks for `python3`, `node`, `npm` dependencies.
- Installs and starts Ollama via Homebrew, pulls `llama3.2:3b`.
- Sets up Python venv, installs all requirements (including CrewAI).
- Starts API (uvicorn), seeds SQLite, starts frontend (Vite dev server).
- Opens browser automatically. Ctrl+C cleans all session artifacts.

#### `run-swarm-fedora.sh` — Fedora Docker Swarm Production
- Installs Docker CE from official Fedora repo if missing.
- Initializes Docker Swarm on the host's primary IP.
- Builds all 5 custom images locally.
- Deploys full stack via `docker stack deploy`.
- Pulls LLM model into the Ollama container post-deploy.

### 4. Environment-Aware Configuration
- **Database**: `session.py` reads `DATABASE_URL` env var — SQLite for local dev, PostgreSQL for Docker.
- **LLM**: `geopolitical_crew.py` reads `OLLAMA_BASE_URL` and `OLLAMA_MODEL` env vars.
- **Frontend**: `api.ts` reads `VITE_API_URL` for build-time API endpoint config.
- **CORS**: Added port 8080 (Docker nginx) to allowed origins.

### 5. Nginx Reverse Proxy
- `docker/nginx.conf` serves the built React SPA and proxies `/api/` requests to the backend service, eliminating CORS issues in production.

### 6. New Dockerfiles
- `docker/api.Dockerfile` — Python 3.11-slim, uvicorn entrypoint.
- `docker/agents.Dockerfile` — Python 3.11-slim, CrewAI + agents deps.
- `docker/ingestion.Dockerfile` — Python 3.11-slim, ingestion processor.
- `docker/frontend.Dockerfile` — Multi-stage: Node 20 build → nginx alpine.
- `docker/seed.Dockerfile` — Python 3.11-slim, one-shot seed_db.py.
